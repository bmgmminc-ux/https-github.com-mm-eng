## 1. 종합 진단

v23은 **뼈대는 이미 실무급인데, 표면과 배관이 베타에 머물러 있는 상태**다. 잘 된 것은 분명하다 — 지표 공식 통일 작업, 거래처 선등록·직불 정책, 툼스톤 삭제 전파, 출역 캘린더의 "데이터→DOM 생성" 렌더러(L13668, 전체의 모범), 디자인 토큰·SVG 아이콘·다크 토큰 세트 같은 자산, brotli 230KB의 양호한 전송. 문제는 **만들어 놓은 자산을 안 쓰거나(토큰 우회 hex 2,025회 vs var() 994회), 만들어 놓은 차트를 숨기고(hideOverviewDemo가 실데이터 배선된 게이지·도넛을 통째 display:none), 가짜를 지우는 대신 런타임 스위퍼로 세탁하는(FAKE_RE + MutationObserver) 구조**다. 실무급 격상의 관건은 4가지다: ① **데이터 안전** — pull 1000행 절단→역push 소실 경로, 전량 delete→insert 동기화, 미저장 편집 침묵 유실. ② **숫자 신뢰** — 가짜 분석 카드가 실데이터와 한 화면에 공존, 같은 게이지에 두 공식이 경쟁, 기타수익 VAT 과대계상. ③ **시간 축** — 월 마감·월별 집계 개념 부재(단, doc_date 필드가 이미 있어 신설이 아니라 연결 문제). ④ **단일 소스** — num() 21중 정의, 거래처 3키 분열 같은 복붙 표류의 종식. 전부 무빌드·단일 HTML 안에서 해결 가능하다.

## 3. 업그레이드 방향 — 7차원

### 디자인·레이아웃
1. **데모 마크업 완전 청산** — 정적 가짜 수치를 0/—로 직접 교체하고 스위퍼 레이어(#150/#151) 은퇴. 렌더 함수만 숫자를 쓰는 단일 경로. 첫 커밋은 탭 제목 '통합 데모' 제거.
2. **현장 모바일 셸** — 하단 탭바(전체현황·프로젝트·출역·매입·더보기) + 640px 이하 카드형 테이블. 출역·영수증 OCR은 본질이 현장(폰) 업무 — 지도·현장공무 축도 이 셸 위에 얹힘.
3. **토큰 강제 로드맵** — 상위 빈도 hex 20종 기계 치환(브랜드 블루 #2563EB/#3B68F6 단일화)→4대 컴포넌트(.btn/.data-table/.form-field/카드) 통일→배포 전 체크에 hex·inline-style 상한 카운터(mm-harness 편입). 완료 전까지 다크 토글 숨김.

### 성능
1. **동기화 프로토콜 v2** — dirty 프로젝트만 push, 행 단위 uuid+upsert diff, pull은 `.range` 페이지네이션+증분. C1·H11의 공통 뿌리이자 50명 계획의 전제.
2. **mmStore 캐시 계층** — 파싱 1회·구독 기반 렌더. ~30곳 독립 JSON.parse를 화면 단위로 순차 교체, 완료 후 3초 폴링 워처 은퇴. 향후 mmPerm(RBAC) 주입 지점 겸용.
3. **외부 의존 전면 lazy 규약** — 기존 우편번호 lazy 패턴(L6721)을 mmLoadScript 공용 유틸로 승격. xlsx부터 시작해 카카오맵·OCR·쿠콘까지 표준화 — head 동기 로드로는 3축 추가 시 첫 화면이 수 초대.
4. **성능 계기판** — 설정 화면에 mm-dash-v4 바이트·costItems 행수·push 소요·pull 행수(1000 근접 경고) 상시 노출. 이 앱의 리스크는 점진 저하가 아니라 절벽형.

### 그래프·시각화

## 5. 하지 않을 것

**불가침 제약 준수** — 아래는 검토 대상에서 원천 제외했으며 위 로드맵 전체가 이 안에서 성립한다:
- 프레임워크 전면 재작성·빌드체인 도입 금지(ADR-001). 모든 개선은 단일 HTML 내 구조 정리·지연 로딩·모듈 경계로 한정.
- v1 React IIFE 부분추출 금지(ADR-002).
- 데모/시드/샘플 데이터 생성 영구 금지 — 빈 화면은 정직한 0. (본 보고의 '가짜 수치 청산'은 이 원칙을 표시 계층까지 관철하는 것.)
- 4대보험 기능 부활 금지. 거래처 선등록·직불 정책 재설계 금지 — 개선은 정책 유지 하의 UX 보조(자동완성·행 표시)에 한정.

**검토했으나 기각한 방향**:
1. **다크모드 즉시 정공 수리** — 하드코딩 hex 2,025회 상태에서 다크 스코프를 지금 다 깔면 토큰 캠페인과 이중 작업. 토글 숨김(3줄)으로 파손 노출부터 막고, 토큰 경유율 정리 후 재개방이 총비용이 작다.
2. **가상 스크롤·테이블 라이브러리 도입** — 매입 수천 행 규모에서는 '최근 200행 + 더보기 + 검색 디바운스'의 무빌드 방식으로 충분하며, 목업 상태인 페이지네이션을 실동작으로 승격하는 편이 외부 의존을 늘리지 않는다.
3. **전면 발주(B) 모듈 즉시 신설** — 현재 업무가 사후 매입 기록 중심이므로 지금 만들면 안 쓰는 화면이 된다. cost_item에 착지 필드(orderAmt 수준)만 선제해 두고, 모듈 자체는 다우오피스 이전 시점에 판단한다.
4. **접근성 전면 개보수** — aria 0건 상태의 일괄 수리는 컴포넌트 통일 전에 하면 재작업. label 연결·아이콘 aria-label·모달 dialog 3점만 우선하고, 나머지는 컴포넌트 통일 시 클래스 단위로 부여한다.
